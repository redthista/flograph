# Known issues

## 1. Flaky test-suite crash at process exit (~25% of runs, pre-existing on master): `double free or corruption` / `QBasicTimer::stop ... different thread` abort during garbage collection. Repro: run `tests/test_show_plot_node.py tests/test_show_table_node.py tests/test_view_keyboard.py` together a few times (offscreen). Involves jedi completion threads (spawned via the editor dock in MainWindow tests) still running while Python GC tears down Qt object cycles; tests themselves all pass — the crash is in teardown. Likely fix directions: shut down jedi/completion threads in MainWindow close, or break Python reference cycles through QWidgets so their teardown doesn't depend on the GC.
