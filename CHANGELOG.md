# Changelog

## 0.1.8

| Category | Change | Details |
| --- | --- | --- |
| UI | Node colours are muted like frames | A custom node colour is now laid over the theme's card colours instead of painted raw, so picker colours come out muted rather than garish — the treatment frame headers and page tabs already use. Applies everywhere a node's colour shows: card body and header, the zoomed-out simplified card, note and Goto/From cards, and the minimap. The stored colour is unchanged, so existing projects keep their colours and just render calmer, and the colour picker still reopens on the colour you chose. |
| Feature | Colour muting strength is configurable | **Settings > Canvas > Custom colour strength** sets how much of a picked colour comes through, as two percentages: one for large surfaces (node card body, unselected page tab) and one for the smaller strip that stands out from it (node header, selected tab). Changes apply live across the canvas, minimap and page tabs, and persist across sessions. 100% paints the colour raw, i.e. no muting at all. |
| Feature | Dashboard page tab colours | Right-click a page tab for **Change colour…** / **Reset colour**, saved with the project and undoable. The colour is laid over the themed tab at low alpha rather than painted flat — the same muting frame headers use — so picker colours come out muted instead of garish, and the selected tab carries a stronger tint so selection still reads. Duplicating a page carries its colour to the copy. |
| Feature | Reorderable dashboard page tabs | Drag a page tab along the bar to reorder the pages; the new order is saved with the project and is a single undoable step. **Model** stays pinned first and the **+** button pinned last, so neither can be dragged away or dropped past. Undoing a page delete now also puts the page back in its original position rather than at the end. |

## 0.1.7

| Category | Change | Details |
| --- | --- | --- |
| Bug Fix | AI Assistant Settings crash fix | Fixed a `NameError` when opening the AI Assistant Settings dialog. |
| Bug Fix | Folium/branca maps in web-view nodes | Rendered via their standalone document instead of the Jupyter-only `_repr_html_()` iframe, which left a stuck "Make this Notebook Trusted to load map" placeholder over the map. |
| Bug Fix | Web-view nodes couldn't load CDN scripts | The embedded webview loads content from a local file, and Qt WebEngine blocks local content from fetching remote resources by default — so a folium/Leaflet map (or anything else pulling JS from a CDN) loaded but never ran. `LocalContentCanAccessRemoteUrls` is now enabled on the view. |
| Bug Fix | Scroll wheel over a web-view card zoomed the canvas | Hovering a folium/Leaflet map (or any other web-view card) and scrolling now lets the embedded map handle its own zoom/pan, instead of the wheel tick always zooming the graph canvas. |
| Bug Fix | One-file build script filename | `scripts/build_onefile_0_1_6.py` renamed to the unversioned `scripts/build_onefile.py` that the README, the script's own docstring, and `tests/test_onefile.py` all already expected — the test was silently failing. |
| Bug Fix | Node Library dock minimum width | Fixes the panel loading too thin, including for layouts saved before this fix existed. |
| Bug Fix | Settings dialog resize | Fixed Canvas page text being clipped top/bottom. |
| Bug Fix | Opening a project with a large cache froze the window | Restoring cached node outputs unpickles one blob per cached node from disk; with many cached nodes this could block the GUI thread long enough to look hung. That work now runs on a background thread with a progress dialog ("Restoring cached results…"), and a second open/new-project attempt while it's running is refused with a status-bar message rather than racing it. A single node with an extremely large cached value can still cause a multi-second stutter (unpickling one blob is not itself chunked), but the common many-node case no longer blocks at all. |
| Bug Fix | Code editor completion ignored Enter/Tab | Accepting a completion suggestion with Enter or Tab inserted a newline/indent instead of the suggested code: the editor's own Return (auto-indent) and Tab (indent) handling consumed the key before the completion popup could act. Popup keys are now intercepted first, and the top suggestion is pre-highlighted so a bare Enter/Tab accepts it. |
| Bug Fix | Canvas minimap was hidden | The minimap was still created, but the opt-in GPU viewport swap landed a new viewport widget on top of it in stacking order, so it looked lost with no way back. The z-order is fixed, and the minimap got its own show/hide toggle in **Settings > Canvas**. |
| Example | Folium Map | New bundled example (File > Open Example) demonstrating a web-view node that returns a raw `folium.Map` object directly — showing off the 0.1.7 fix that lets the webview card unwrap folium/branca objects without calling `get_root().render()` by hand. |
| Feature | Configurable page bar position | Model/page tab strip can be pinned to the top or bottom of the window (Settings > General), persisted across sessions. |
| Feature | Dashboard Visuals panel toggle | Collapse/expand the Visuals panel on dashboard pages to give the canvas full width when it's not needed. |
| Feature | Disable SSL verification for AI requests | New "Verify SSL certificates" checkbox in AI Assistant Settings, for local/proxied LLM servers with self-signed certs. |
| Feature | Drag-and-drop file readers | Dropping a `.csv`/`.xlsx`/`.xls`/`.xlsm`/`.parquet` file from the OS file explorer onto the canvas creates the matching reader node with its path pre-filled. |
| Feature | Join node overhaul | The default Join node now supports the full range of pandas `join` features. |
| Feature | Password param type | Node scripts can declare `{"type": "password"}` in `PARAMS` for a masked field with a Show/Hide toggle. |
| Feature | Canvas preview toggle | Right-click a figure/webview/table/slicer node for a "Disable/Enable Canvas Preview" action — hides the embedded widget on the model canvas to save render cost while the node still renders fully on Dashboard pages. |
| Feature | About page in Settings | New **Settings > About** page shows the installed flograph version (read live from package metadata), plus Python/Qt versions. |
| Feature | Table node tiles on dashboard pages | The IO > Table node (the manually-editable spreadsheet source) can now be dragged onto Dashboard pages like the other visuals, showing its output DataFrame as a live read-only table. |
| Feature | Reader node overhaul | All five reader nodes (CSV, Excel, JSON, Parquet, SQLite) now expose the full pandas option set. CSV: separator with `\t` and auto-sniff, encoding, column selection, index column, skip/max rows, decimal/thousands/quote/comment characters, extra missing values, date parsing, per-column dtype overrides, blank-line/bad-line handling, and parser engine choice. Excel: sheet by name/index or `*` to stack every sheet with a `sheet` column, Excel letter ranges (`A:C,F`), skip/max rows, dtype overrides, and engine choice. JSON: encoding, nested-object flattening (`json_normalize`) with configurable separator, JSONL row limit, date detection toggle. Parquet: pushdown row filters (`column op value` per line), engine and dtype-backend choice. SQLite: query or whole-table mode (a blank table name lists the tables in the database), index column, date parsing, dtype overrides. Existing saved graphs keep their old params and behave identically. |
| Feature | Writer node overhaul | All five writer nodes now expose the full pandas option set, matching the reader overhaul. CSV: separator (`\t` supported), header toggle, append mode that skips repeating the header, encoding, column subset, missing-value text, float/date formats, decimal mark, and quoting style; a compressed extension (.gz/.zip/.bz2/.xz/.zst) compresses automatically. Excel: 'add sheet' writes into an existing workbook without clobbering its other sheets (replace/error/new/overlay on collision), header toggle, start row/column offsets, freeze header row, missing-value text, float format, and engine choice. JSON: ISO or epoch dates with configurable precision, float digits, and a non-ASCII escape toggle. Parquet: compression choice (snappy/gzip/brotli/zstd/lz4/none) and hive-style partition columns whose folders the Read Parquet row filters can skip. SQLite: per-column SQL type overrides, index column name, and batched writes. Existing saved graphs keep their old params and behave identically. |
| Feature | Table node is now a real spreadsheet | The IO > Table node grew from a plain grid into an Excel-familiar spreadsheet. Cells starting with `=` are formulas: A1-style references with `$` pins, named column references (`[@Price]` for this row's value, `[Price]` for the whole column — `=[@Price]*[@Qty]`, `=SUM([Total])`; names may contain spaces, match case-insensitively, follow column renames automatically, and never shift on paste/fill-down or when columns move), the usual operators, and 25+ functions (SUM, AVERAGE, IF, ROUND, CONCAT, …) with dependency-ordered recalculation, cycle detection, and Excel-style error codes shown in the cell with a tooltip. Columns have types (auto/text/number/integer/date/bool) set from the header menu — entries that don't fit are flagged red as you type, bools become checkboxes, dates get a calendar editor, and the output DataFrame gets proper dtypes. Copy/paste is Excel-grade: rectangular selections copy as TSV + HTML (paste cleanly into Excel/Sheets) while in-app paste keeps formulas and shifts their relative references; pasting a bigger block auto-grows the grid, and one copied cell replicates across a selection. Header menus insert/delete rows and columns anywhere and sort; Ctrl+D fills down; Enter/Tab/F2/Delete behave like Excel. Formula entry autocompletes — function names after two letters (inserted with their opening paren) and column names as soon as you type `[` or `[@`, in both cell editors and the formula bar; Enter/Tab accept the highlighted suggestion, Escape dismisses. The card's new expand button opens a full editor window with a formula bar (its fx button opens a formula reference listing every function with an example) and its own fine-grained undo, committing back to the canvas as a single undo step; Enter in the editor commits the cell and moves down — it never "clicks" OK and closes the window — while on-card edits are now one undo step per edit instead of all merging together. Column widths are saved with the node; a Fit button on the card and editor, header-menu "Resize to content" options, and border double-click (fitting the whole selection) auto-size columns; **Settings > Table Node** auto-fits after every edit (on by default — turn it off to use saved manual widths). Switching a column to the date type auto-converts recognisable formats ("23/07/2026", "Jul 5, 2026", "07-Mar-12", …) to ISO, flagging only what it can't read, and Settings > Table Node accepts extra strptime patterns (e.g. `%d-%b-%y`) that take priority over the built-ins. The row-number menu can promote a row to the column headers. Existing saved graphs load unchanged, and the raw-JSON "Table data" box is gone from the Properties panel. |
| Feature | Table node linked input | The Table node gained an optional dataframe input, modeled on an Excel table with custom columns beside a Power Query load. While connected, every run refreshes the columns the input owns (matched by name; dtypes become column types, dates arrive ISO, missing values become blank cells) — but the grid stays fully editable, and columns you add on top survive each refresh: as rows grow, a trailing formula fills down with shifted references; as rows shrink, extra cells trim off. Your saved column widths also survive refreshes. Edits inside input-owned columns are overwritten on the next run — put your work in your own columns. Right-click the node for **Import input into table** to snapshot the merged result into the grid as one undoable step; disconnect the input and it's fully yours. Unconnected, the node behaves exactly as before. |
| Feature | Node colours | Right-click a node for "Change colour…" / "Reset colour" — the custom header colour is saved with the project and reflected in the minimap. |
| Feature | Reroute labels and descriptions | Reroutes stay dot-only by default; naming one (Rename or the Properties panel) shows the label as a pill above the dot, and a new Description field shows as a hover tooltip (yielding to the error tooltip when the node is in an error state). Also fixes a pre-existing bug where opening Rename and clicking OK unedited silently turned the default label into an explicit override. |
| Feature | Copy/paste for frames | Frames join the same clipboard flow as nodes: copying a frame also carries the nodes sitting inside it (matching drag behavior), and pasting recreates both. Right-click on a node or frame now offers "Copy", and right-clicking empty canvas offers "Paste" whenever the clipboard holds a valid payload. |
| Feature | Goto / From links | New **Util > Goto** and **Util > From** nodes carry a value across the canvas with no wire between them — Simulink's goto/from, for keeping a busy model readable. Drop a Goto, name the link, then drop From nodes anywhere and pick that link from their **Goto** dropdown; each one re-emits whatever the Goto received. A From binds to the Goto *node*, not to its name, so renaming a link renames it at both ends immediately, renaming can never break a link, and two links may share a name. The engine treats a link exactly like a wire: execution order, dirty propagation, cached-output invalidation, "run to here" and headless runs all follow links, and a link that would close a loop is refused with an explanation rather than breaking the run. Since there's no wire to trace, selecting either end glows the other, and a From that points at nothing (or at a Goto that has since been deleted) turns red and stops the run instead of passing an empty value downstream. Copying a Goto/From pair rewires the copies to each other; copying a lone From keeps it reading the original Goto. |
| Performance | GPU-accelerated canvas + zoom LOD | New **Tools > Settings…** (Ctrl+,) dialog; opt-in OpenGL viewport (auto-reverts if GL isn't available) and zoom-out node simplification to keep large graphs responsive. |
| UI | Canvas zoom indicator | The status bar's bottom-right corner shows the zoom level live (updating as you wheel-zoom or fit); clicking it snaps back to 100%, keeping the view centred on the same spot. It follows whichever page is showing — the model canvas or the active dashboard page, each of which keeps its own zoom. |
| UI | Code editor header cleanup | "Ask AI…", "Save as user node…", and "Reset to library" moved from the header down to the footer, next to Apply; the node title now stretches to fill the available width and elides with a tooltip instead of truncating at a fixed size. |
| UI | Properties panel redesigned as a resizable table | Node parameters now show in a two-column Property/Value table instead of an auto-generated form — drag the column divider to resize, so long labels or params never force the panel wider. |
| UI | Settings window reorganized | Snap-to-Grid and grid resolution moved from the toolbar into **Settings > Canvas**; page bar position got its own new **Settings > General** page. |
| UI | Snap-to-Grid on by default | Previously off by default; now enabled out of the box (toggle and grid resolution still configurable in Settings). |
| UI | Settings nav sorted alphabetically | The Settings category list (General/Canvas/About, etc.) now sorts ascending instead of insertion order. |
| UI | Dashboard tile close button removed | Tiles no longer show a red X in the corner — right-click to select, then press Delete, same as every other tile-removal path. The title bar text now uses the full width instead of leaving a gap where the button was. |

| UI | Frame resize from any edge | Comment frames can be resized by dragging the right, bottom, or left edge, not just the bottom-right corner — matching how nodes and dashboard tiles already behave. |
